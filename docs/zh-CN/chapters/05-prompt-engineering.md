# 05. 智能体提示词工程

> **本章内容：智能体提示词和聊天提示词有什么不同？如何写出让智能体表现更好的系统提示词。**

---

## 🎯 目标

学完本章，你将能够：
- 理解智能体提示词的特殊之处
- 写出清晰、有效的系统提示词
- 动态构建提示词（注入工具列表、上下文）
- 用 few-shot 示例提高准确率

---

## 📖 给 10 岁孩子解释

提示词就像给智能体的"说明书"。

普通聊天提示词："帮我翻译这句话。"
智能体系统提示词：一份详细的员工手册。

好的系统提示词让智能体知道：
- 你是谁（角色）
- 你能做什么（能力）
- 你怎么做（流程）
- 你不能做什么（限制）

## 🔧 动手实践

### Step 1：基础系统提示词

```python
SYSTEM_PROMPT = """你是一个智能助手，可以使用以下工具：

{tool_descriptions}

使用规则：
1. 先思考用户需要什么
2. 如果需要工具，使用工具
3. 把工具结果用易懂的语言告诉用户
4. 如果工具出错了，尝试别的方法
5. 不知道就说不知道

回复要简洁、准确。"""
```

### Step 2：动态提示词构建

```python
def build_system_prompt(registry: ToolRegistry, user_name: str = None) -> str:
    """根据当前状态动态构建系统提示词。"""
    parts = [
        "你是一个智能体助手。",
        "",
        "## 可用工具",
    ]
    for tool in registry.get_all_tools():
        parts.append(f"- **{tool.name}**：{tool.description}")

    parts.extend([
        "",
        "## 工作方式",
        "- 一步一步思考",
        "- 需要时使用工具",
        "- 把工具结果转化成易懂的回答",
        "- 完成后告诉用户",
    ])

    if user_name:
        parts.extend(["", f"## 当前用户", f"你正在和 {user_name} 对话。"])

    parts.extend([
        "",
        f"当前时间：{datetime.now().strftime('%Y-%m-%d %H:%M')}",
    ])

    return "\n".join(parts)
```

### Step 3：不同角色的提示词

```python
RESEARCHER_PROMPT = """你是研究员智能体。
你的工作是：搜索信息、阅读资料、整理发现。
你要提供准确的引用和来源。
不要编造数据。如果找不到就说找不到。"""

WRITER_PROMPT = """你是写作智能体。
你的工作是：基于研究结果写文章。
你擅长把复杂概念说得简单易懂。
风格：清晰、生动、有条理。"""

CRITIC_PROMPT = """你是审查智能体。
你的工作是：检查作品的问题。
检查：事实错误、逻辑矛盾、遗漏信息。
提供具体的改进建议。"""
```

### Step 4：Few-Shot 示例

在提示词中加入例子，让 LLM 更好地理解你的意图：

```python
FEW_SHOT_EXAMPLES = [
    {
        "user": "法国面积是多少？",
        "assistant": "让我查一下。\nACTION: {\"tool\": \"search\", \"args\": {\"query\": \"法国面积\"}}",

```

## 🔍 提示词设计原则

| 原则 | 不好 | 好 |
|---|---|---|
| 具体 | "做好你的工作" | "搜索信息并用中文总结" |
| 可验证 | "高质量回答" | "引用来源，附上链接" |
| 边界清晰 | "能做任何事" | "只能用以下工具..." |
| 错误处理 | "不要出错" | "如果工具失败，告诉用户并尝试替代方案" |

## 🧪 练习

1. 写三份不同人格的提示词（严谨教师、幽默朋友、专业顾问），测试同一问题
2. 在提示词中加入"回复不超过 3 句话"规则，测试效果
3. 添加"如果你不确定，就说'我需要查一下'，然后用搜索工具"规则

## 📝 总结

- 智能体提示词 = 说明书，定义了角色、能力、流程
- 动态构建提示词让智能体了解当前上下文
- Few-shot 示例大幅提高工具调用准确率
- 测试不同提示词，找出最适合你场景的
